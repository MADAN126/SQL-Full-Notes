# 20. SUB QUERIES

## Introduction

A Sub Query is a query written inside another query.

Also known as:

- Nested Query
- Inner Query
- Query within Query

Subqueries are used when one query depends on the result of another query.

---

## Why Use Sub Queries?

### Scenario

Suppose you want employees working in the SALES department.

You don't know the department number of SALES.

Step 1: Find SALES department number.

```sql
SELECT deptno
FROM dept
WHERE dname = 'SALES';
```

Step 2: Use that result to get employees.

```sql
SELECT *
FROM emp
WHERE deptno = 30;
```

Subquery combines both:

```sql
SELECT *
FROM emp
WHERE deptno =
(
    SELECT deptno
    FROM dept
    WHERE dname='SALES'
);
```

---

## Execution Flow

Subqueries execute from inside to outside.

```text
Inner Query
     ↓
Result
     ↓
Outer Query
     ↓
Final Output
```

Example:

```sql
SELECT *
FROM emp
WHERE sal =
(
    SELECT MAX(sal)
    FROM emp
);
```

Execution:

```text
MAX(sal) = 5000
     ↓
SELECT * FROM emp WHERE sal = 5000
```

---

## General Syntax

```sql
SELECT column_name
FROM table_name
WHERE condition
(
    SELECT column_name
    FROM table_name
);
```

# Types of Sub Queries

## 1. Single Row Sub Query

Returns exactly one row.

### Operators Used

```sql
=
>
<
>=
<=
<>
```

---

### When Does It Return One Row?

#### Case 1: Primary Key Search

```sql
SELECT sal
FROM emp
WHERE empno = 7788;
```

Returns one row because `empno` is primary key.

---

#### Case 2: Aggregate Functions

```sql
SELECT MAX(sal)
FROM emp;
```

Returns one row.

---

### Example

#### Highest Salary Employee

```sql
SELECT *
FROM emp
WHERE sal =
(
    SELECT MAX(sal)
    FROM emp
);
```

---

#### Lowest Salary Employee

```sql
SELECT *
FROM emp
WHERE sal =
(
    SELECT MIN(sal)
    FROM emp
);
```

---

## 2. Multi Row Sub Query

Returns multiple rows.

### Operators Used

```sql
IN
NOT IN
ANY
ALL
EXISTS
NOT EXISTS
```

---

### Example

```sql
SELECT deptno
FROM dept;
```

Returns:

```text
10
20
30
40
```

Multiple rows ⇒ Multi Row Sub Query.

---

### Correct Usage

```sql
SELECT *
FROM emp
WHERE deptno IN
(
    SELECT deptno
    FROM dept
);
```

---

# Important Difference

## Single Row

```sql
SELECT *
FROM emp
WHERE sal =
(
    SELECT MAX(sal)
    FROM emp
);
```

Returns one value.

---

## Multi Row

```sql
SELECT *
FROM emp
WHERE deptno IN
(
    SELECT deptno
    FROM dept
);
```

Returns many values.

# Practical Examples

## 1. Department Details of SMITH

### Step 1

Find SMITH's department.

```sql
SELECT deptno
FROM emp
WHERE ename='SMITH';
```

### Step 2

Get department information.

```sql
SELECT *
FROM dept
WHERE deptno IN
(
    SELECT deptno
    FROM emp
    WHERE ename='SMITH'
);
```

---

## 2. Employees Working Under SALES Department

```sql
SELECT *
FROM emp
WHERE deptno =
(
    SELECT deptno
    FROM dept
    WHERE dname='SALES'
);
```

---

## 3. Employees Earning Less Than Employee 7788

```sql
SELECT *
FROM emp
WHERE sal <=
(
    SELECT sal
    FROM emp
    WHERE empno=7788
);
```

---

## 4. Employees Earning Less Than 7788 and Receiving Commission

```sql
SELECT *
FROM emp
WHERE sal <=
(
    SELECT sal
    FROM emp
    WHERE empno=7788
)
AND comm IS NOT NULL;
```

---

## 5. Employees Working Under Manager BLAKE

### Step 1

Find BLAKE's employee number.

```sql
SELECT empno
FROM emp
WHERE UPPER(ename)='BLAKE';
```

### Step 2

Find employees managed by BLAKE.

```sql
SELECT *
FROM emp
WHERE mgr =
(
    SELECT empno
    FROM emp
    WHERE UPPER(ename)='BLAKE'
);
```

---

## 6. Employees Working In CHICAGO

```sql
SELECT *
FROM emp
WHERE deptno =
(
    SELECT deptno
    FROM dept
    WHERE UPPER(loc)='CHICAGO'
);
```

---

## 7. Second Highest Salary

### Highest Salary

```sql
SELECT MAX(sal)
FROM emp;
```

### Second Highest Salary

```sql
SELECT MAX(sal)
FROM emp
WHERE sal <
(
    SELECT MAX(sal)
    FROM emp
);
```

---

### Employee Details With Second Highest Salary

```sql
SELECT *
FROM emp
WHERE sal =
(
    SELECT MAX(sal)
    FROM emp
    WHERE sal <
    (
        SELECT MAX(sal)
        FROM emp
    )
);
```
