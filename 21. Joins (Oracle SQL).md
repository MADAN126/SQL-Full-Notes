# 21. Joins (Oracle SQL)

## Definition

A Join is used to retrieve data from one or more tables based on a common column.

- Parent table contains Primary Key.
- Child table contains Foreign Key.
- The condition connecting tables is called the Join Condition.

Example:

```sql
SELECT *
FROM emp e
JOIN dept d
ON e.deptno = d.deptno;
```

Join Condition:

```sql
e.deptno = d.deptno
```

---

## Why Joins?

Without Join:

```sql
SELECT * FROM emp;
SELECT * FROM dept;
```

Data is retrieved separately.

With Join:

```sql
SELECT e.ename,d.dname
FROM emp e
JOIN dept d
ON e.deptno=d.deptno;
```

Employee and department information are retrieved together.

---

## Physical Join vs Logical Join

### Physical Join

Relationship created using constraints.

```sql
PRIMARY KEY
FOREIGN KEY
```

Example:

```sql
dept.deptno  --> Primary Key
emp.deptno   --> Foreign Key
```

---

### Logical Join

Relationship used while retrieving data.

```sql
SELECT *
FROM emp e
JOIN dept d
ON e.deptno=d.deptno;
```

---

## Types of Joins

2. Left Join

4. Full Outer Join

---

## Inner Join (Equi Join)

### Definition

Returns only matching rows from both tables based on the join condition.

Rows without matching values are ignored.

---

### Syntax

```sql
SELECT columns
FROM table1
INNER JOIN table2
ON table1.column = table2.column;
```

---

### Old Style Syntax

```sql
SELECT columns
FROM table1, table2
WHERE table1.column = table2.column;
```

Not recommended because WHERE clause should ideally be used for filtering.

---

### Recommended Syntax

```sql
SELECT columns
FROM table1
JOIN table2
ON table1.column = table2.column;
```

---

## Important Interview Questions

### What is a Join?

A Join is used to retrieve data from multiple tables using a common column.

---

### What is a Join Condition?

A condition that connects two tables.

Example:

```sql
e.deptno = d.deptno
```

---

### Why are joins required?

To retrieve related data from multiple tables in a single query.

---

### Which columns are commonly used in joins?

```text
Parent Table  -> Primary Key
Child Table   -> Foreign Key
```

---

### Which join returns only matching records?

```text
Inner Join
```

---

### Which syntax is recommended?

```sql
SELECT *
FROM emp e
JOIN dept d
ON e.deptno = d.deptno;
```

because join condition is clearly separated from filtering conditions.

---

## Quick Revision

- Join combines data from multiple tables.
- Common column is mandatory.
- Join condition connects tables.
- Primary Key ↔ Foreign Key relationship is commonly used.
- Inner Join returns only matching rows.
- Modern JOIN ... ON syntax is preferred.
- WHERE clause should mainly be used for filtering.
