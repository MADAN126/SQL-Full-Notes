# SQL Joins & Set Operators

## Self Join

### Definition
- A Self Join is a join where a table is joined with itself.
- Used when data in the same table is related to other rows in the same table.
- Commonly used for:
  - Employee ↔ Manager relationships
  - Parent ↔ Child relationships
  - Hierarchical data

### Syntax (Old Style)

```sql
SELECT a.column_name, b.column_name
FROM table_name a, table_name b
WHERE a.common_field = b.common_field;
```

### Syntax (ANSI Join)

```sql
SELECT a.column_name, b.column_name
FROM table_name a
JOIN table_name b
ON a.common_field = b.common_field;
```

## Employee-Manager Example

### Requirement
Display employee names along with their manager names.

### Query

```sql
SELECT e.ename || ' Works Under Manager ' || m.ename AS Employee_Manager_Name
FROM emp e
JOIN emp m
ON e.mgr = m.empno
ORDER BY e.ename;
```

### Requirement
Display child name along with parent name.

### Query

```sql
SELECT f1.name || ' Parent is ' || f2.name
FROM family f1
LEFT JOIN family f2
ON f1.parent_id = f2.member_id
ORDER BY f2.member_id;
```

### Sample Output

```text

Mahesh Parent is Naresh
Johnson Parent is Naresh
Naresh Parent is Rajesh
Avinash Parent is Suresh
Anil Parent is Suresh
```

# Set Operators

Set Operators combine results of two or more SELECT statements.

## Rules

- Number of columns must be the same in all SELECT statements.
- Data types of corresponding columns must be compatible.
- ORDER BY can be used only at the end.

---

## UNION

### Definition

- Combines results of multiple queries.
- Removes duplicate rows automatically.

### Syntax

```sql
SELECT column_list FROM table1
UNION
SELECT column_list FROM table2;
```

### Example

```sql
SELECT deptno FROM emp
UNION
SELECT deptno FROM dept;
```

### Result

```text
Duplicates Removed
Unique Records Returned
```

---

## UNION ALL

### Definition

- Combines results of multiple queries.
- Includes duplicate records.

### Syntax

```sql
SELECT column_list FROM table1
UNION ALL
SELECT column_list FROM table2;
```

### Example

```sql
SELECT deptno FROM emp
UNION ALL
SELECT deptno FROM dept;
```

### Result

```text
Duplicates Included
Faster than UNION
```

---

## INTERSECT

### Definition

- Returns only common records present in both queries.

### Syntax

```sql
SELECT column_list FROM table1
INTERSECT
SELECT column_list FROM table2;
```

### Example

```sql
SELECT deptno FROM emp
INTERSECT
SELECT deptno FROM dept;
```

### Result

```text
Only Matching/Common Records
```

---

## MINUS

### Definition

- Returns records present in the first query but not in the second query.

### Syntax

```sql
SELECT column_list FROM table1
MINUS
SELECT column_list FROM table2;
```

### Example

```sql
SELECT deptno FROM emp
MINUS
SELECT deptno FROM dept;
```

### Result

```text
Records existing only in first query
```

---

# Quick Comparison

| Operator | Duplicates | Purpose |
|-----------|------------|----------|
| UNION | Removed | Combine unique rows |
| UNION ALL | Kept | Combine all rows |
| INTERSECT | Removed | Common rows only |
| MINUS | Removed | First query rows not in second query |

# Real-Time Usage

### Self Join
- Employee → Manager
- Parent → Child
- Category → Parent Category

### UNION
- Active customers + New customers

### UNION ALL
- Merge large datasets without removing duplicates

### INTERSECT
- Common customers between two products

### MINUS
- Customers who bought Product A but not Product B
