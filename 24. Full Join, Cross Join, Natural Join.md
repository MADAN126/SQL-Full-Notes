# SQL Joins – Full Join, Cross Join, Natural Join

## Full Join (Full Outer Join)

### Definition
A Full Join returns:

- All matching records from both tables.
- Unmatched records from the left table.
- Unmatched records from the right table.

### Formula

```sql
FULL JOIN =
INNER JOIN
+ Unmatched rows from Left Table
+ Unmatched rows from Right Table
```

### Syntax

```sql
SELECT columns
FROM table1
FULL JOIN table2
ON table1.column = table2.column;
```

### Example

```sql
SELECT ae.emp_id,
       ae.emp_name,
       ad.dept_name,
       ae.dept_id,
       ad.dept_id
FROM ashokit_emp ae
FULL JOIN ashokit_dept ad
ON ae.dept_id = ad.dept_id
ORDER BY ae.emp_id;
```

### Result Analysis

#### Matched Records (Inner Join Result)

```text
E1  Anil         SALES
E2  Avinash      IT
E3  Raju         HR
E4  Raghavendra  ADMIN
E5  Abdul        MARKETING
E6  Gupta        IT
E7  Bhavya       SALES
```

#### Unmatched Records from Left Table

```text
E8  SUNIL  D11
E9  ABHI   D12
```

Department not found, so right table columns become NULL.

#### Unmatched Records from Right Table

```text
D3  FINANCE
D5  NON-ADMIN
```

No employees belong to these departments, so employee columns become NULL.

### Important Points

- Returns all records from both tables.
- Useful when missing data from either side must be visible.
- Commonly used for auditing and reconciliation reports.

## Cross Join (Cartesian Product)

### Definition

Cross Join combines every row of the first table with every row of the second table.

### Formula

```text
Rows Returned = Rows in Table A × Rows in Table B
```

### Characteristics

- No join condition required.
- Every left row pairs with every right row.

### Syntax

```sql
SELECT *
FROM table1
CROSS JOIN table2;
```

### Example

```sql
SELECT ae.emp_name,
       COALESCE(ad.dept_name,'NO-DEPARTMENT') AS dname,
       ci.company_name,
       ci.company_location
FROM ashokit_emp ae
LEFT JOIN ashokit_dept ad
ON ae.dept_id = ad.dept_id
CROSS JOIN companies_info ci
ORDER BY ae.emp_name;
```

### Sample Output

```text
ABHI          NO-DEPARTMENT  ASHOKIT  HYDERABAD
Abdul         MARKETING      ASHOKIT  HYDERABAD
Anil          SALES          ASHOKIT  HYDERABAD
Avinash       IT             ASHOKIT  HYDERABAD
Bhavya        SALES          ASHOKIT  HYDERABAD
Gupta         IT             ASHOKIT  HYDERABAD
Raghavendra   ADMIN          ASHOKIT  HYDERABAD
Raju          HR             ASHOKIT  HYDERABAD
SUNIL         NO-DEPARTMENT  ASHOKIT  HYDERABAD
```

### Important Points

- Also called **Cartesian Product**.
- Does not use `ON` condition.
- Can generate huge result sets.
- Use carefully in production systems.

## Natural Join

### Definition

Natural Join automatically joins tables using columns that have the same name.

### Characteristics

- No need to specify join condition.
- Oracle automatically finds common column names.
- Behaves like an Inner Join using those columns.

### Syntax

```sql
SELECT columns
FROM table1
NATURAL JOIN table2;
```

### Example

```sql
SELECT ae.emp_id,
       ae.emp_name,
       ad.dept_name
FROM ashokit_emp ae
NATURAL JOIN ashokit_dept ad
ORDER BY ae.emp_id;
```

### Output

```text
E1  Anil         SALES
E2  Avinash      IT
E3  Raju         HR
E4  Raghavendra  ADMIN
E5  Abdul        MARKETING
E6  Gupta        IT
E7  Bhavya       SALES
```

### Important Points

- Oracle automatically uses common columns (`dept_id`).
- No need for `ON` clause.
- If no common column exists, Oracle performs a Cartesian Product.
- Rarely used in real-world projects because explicit joins are clearer and safer.

# Join Summary

| Join Type | Returns |
|------------|----------|
| INNER JOIN | Only matching rows from both tables |
| LEFT JOIN | Matching rows + unmatched rows from left table |
| RIGHT JOIN | Matching rows + unmatched rows from right table |
| FULL JOIN | Matching rows + unmatched rows from both tables |
| CROSS JOIN | Every row from left table with every row from right table |
| NATURAL JOIN | Automatically joins using common column names |
| SELF JOIN | A table joined with itself |

---

# Real-Time Usage Priority

Most commonly used joins in projects:

1. INNER JOIN
2. LEFT JOIN
3. RIGHT JOIN
4. FULL JOIN
5. SELF JOIN

Less commonly used:

- CROSS JOIN
- NATURAL JOIN

# Key Interview Questions

### What is Full Join?

Returns matched rows plus unmatched rows from both tables.

### What is the difference between Left Join and Full Join?

```text
LEFT JOIN  -> Left unmatched rows only
FULL JOIN  -> Left unmatched + Right unmatched rows
```

### What is Cross Join?

Every row of one table is combined with every row of another table.

### What is Cartesian Product?

Another name for Cross Join.

### Does Cross Join need an ON condition?

No.

### What is Natural Join?

Oracle automatically creates the join condition using common column names.

### Why is Natural Join rarely used?

Because automatic join conditions can cause unexpected results and reduce query readability.

### Which joins are used most in real projects?

```text
INNER JOIN
LEFT JOIN
RIGHT JOIN
FULL JOIN
SELF JOIN
```
