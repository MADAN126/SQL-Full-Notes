# 19. GROUP BY & HAVING CLAUSE

## GROUP BY

### Definition
`GROUP BY` is used to group rows having the same values into summary groups.

It is mainly used with aggregate functions:

- COUNT()
- SUM()
- AVG()
- MIN()
- MAX()

Each group returns one result row.

---

## Syntax

```sql
SELECT column_name, aggregate_function(column_name)
FROM table_name
GROUP BY column_name;
```

### Example

```sql
SELECT deptno, COUNT(*)
FROM emp
GROUP BY deptno;
```

---

## Rules of GROUP BY

### Rule 1

All non-aggregate columns in SELECT must appear in GROUP BY.

✅ Correct

```sql
SELECT deptno, COUNT(*)
FROM emp
GROUP BY deptno;
```

❌ Wrong

```sql
SELECT deptno, ename
FROM emp
GROUP BY deptno;
```

---

### Rule 2

WHERE executes before GROUP BY.

```sql
SELECT deptno, COUNT(*)
FROM emp
WHERE sal > 1000
GROUP BY deptno;
```

---

## Common GROUP BY Queries

### Number of Employees in Each Department

```sql
SELECT deptno, COUNT(*)
FROM emp
GROUP BY deptno;
```

---

### Maximum Salary in Each Department

```sql
SELECT deptno, MAX(sal)
FROM emp
GROUP BY deptno;
```

---

### Employee Count Job Wise

```sql
SELECT job, COUNT(*)
FROM emp
GROUP BY job;
```

---

### Salary Statistics Job Wise

```sql
SELECT job,
       MIN(sal),
       MAX(sal),
       SUM(sal),
       AVG(sal),
       COUNT(*)
FROM emp
GROUP BY job;
```

---

### Employees Having Salary > 2500

```sql
SELECT deptno,
       sal,
       COUNT(*)
FROM emp
WHERE sal > 2500
GROUP BY deptno, sal;
```

---

# HAVING CLAUSE

## Definition

HAVING is used to filter grouped data.

- WHERE filters rows
- HAVING filters groups

HAVING is usually used with GROUP BY.

---

## Syntax

```sql
SELECT column_name,
       aggregate_function(column_name)
FROM table_name
GROUP BY column_name
HAVING condition;
```

---

## WHERE vs HAVING

| WHERE | HAVING |
|---------|---------|
| Filters rows | Filters groups |
| Before GROUP BY | After GROUP BY |
| Cannot use aggregates directly | Can use aggregates |
| Row-level filtering | Group-level filtering |

---

## Example

### WHERE

```sql
SELECT *
FROM emp
WHERE sal > 2000;
```

Filters employees.

---

### HAVING

```sql
SELECT deptno, COUNT(*)
FROM emp
GROUP BY deptno
HAVING COUNT(*) > 3;
```

Filters departments.

---

## Common HAVING Queries

### Departments Having At Least 5 Employees

```sql
SELECT deptno,
       COUNT(*) AS NoOfEmployees
FROM emp
GROUP BY deptno
HAVING COUNT(*) >= 5;
```

---

### Jobs Having Employee Count <= 3

```sql
SELECT job,
       COUNT(*) AS NoOfEmployees
FROM emp
GROUP BY job
HAVING COUNT(*) <= 3;
```

---

### Departments Having Minimum Salary Less Than 1000

```sql
SELECT deptno,
       MIN(sal),
       MAX(sal),
       SUM(sal)
FROM emp
GROUP BY deptno
HAVING MIN(sal) < 1000;
```

---

### Departments Having Maximum Salary Greater Than 2500

```sql
SELECT deptno,
       MIN(sal),
       MAX(sal),
       SUM(sal)
FROM emp
GROUP BY deptno
HAVING MAX(sal) > 2500;
```

---

# Execution Order of SQL Clauses

```text
FROM
 ↓
WHERE
 ↓
GROUP BY
 ↓
HAVING
 ↓
ORDER BY
 ↓
SELECT
```

---

# Interview Questions

## Why do we use GROUP BY?

To create groups and perform aggregate calculations on each group.

---

## Difference Between WHERE and HAVING?

### WHERE

- Filters individual rows
- Executes before grouping

### HAVING

- Filters grouped results
- Executes after grouping

---

## Can HAVING Be Used Without GROUP BY?

Yes.

```sql
SELECT COUNT(*)
FROM emp
HAVING COUNT(*) > 10;
```

---

## Can GROUP BY Use Multiple Columns?

Yes.

```sql
SELECT deptno,
       job,
       COUNT(*)
FROM emp
GROUP BY deptno, job;
```

Groups records based on both department and job.

---

# Quick Revision

## GROUP BY

- Creates groups
- Used with aggregate functions
- Returns one row per group

## HAVING

- Filters groups
- Used after GROUP BY
- Works with aggregate functions

## Aggregate Functions

```sql
COUNT()
SUM()
AVG()
MIN()
MAX()
```

